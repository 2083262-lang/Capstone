HomeEstate Realty — User Manual Section 1: Introduction

1. Introduction

System background, manual purpose, and intended audience

1.1 Purpose of the System

HomeEstate Realty is a web-based real estate property management system designed to streamline the processes of listing, marketing, selling, and renting real estate properties in the Philippines. The system serves as a centralized platform that connects property administrators, licensed real estate agents, and prospective clients (buyers and tenants) in a unified digital environment.

The platform supports the complete property lifecycle — from initial listing creation by agents or administrators, through administrative approval, to the finalization of sales and rental agreements. It incorporates comprehensive features including property search and filtering, image gallery management, tour scheduling, rental payment tracking, agent commission processing, and automated email notifications.

HomeEstate Realty is developed using PHP and MySQL, deployed on an Apache web server environment (XAMPP), and implements session-based authentication with two-factor authentication (2FA) via email-delivered one-time passwords (OTP), ensuring secure access for all authorized personnel.

1.2 Target Audience

This manual is written for three categories of system users:

👤 Administrator

Manages the entire platform including property approvals, agent management, sale/rental finalization, commission processing, and system configuration.

🏢 Real Estate Agent

Licensed professionals who list properties, manage tours, process sales and rentals, record payments, and track commissions.

🌐 Public User / Client

Prospective buyers, tenants, or visitors who browse property listings, view agent profiles, and request tours — no account required.

1.3 Scope of the Manual

This manual covers all features and functionalities of the HomeEstate Realty system as deployed in version 1.0. It includes step-by-step instructions for account registration, login, property management, tour scheduling, sales and rental processing, payment recording, commission tracking, system administration, and public browsing. The document also provides troubleshooting guidance, a glossary of terms, and best practices for each user role.

HomeEstate Realty — User Manual Section 2: System Overview

2. System Overview

Architecture, features, and functional modules

2.1 General Description

HomeEstate Realty operates as a three-tier web application consisting of a public-facing portal for property browsing, an agent portal for property management, and an administrative portal for oversight and approvals. The system enforces a structured approval workflow ensuring that all property listings, sales, and rental transactions are verified by the administrator before finalization.

Key Architectural Characteristics

Role-Based Access Control (RBAC) — Three distinct user roles with granular permission enforcement.

Approval-Based Workflow — All submissions require administrative review before finalization.

Two-Factor Authentication (2FA) — Mandatory OTP verification for admin and agent logins.

Email Notification System — Automated alerts via PHPMailer for all key platform events.

Session Security — 30-minute timeout, secure cookies, session regeneration, and IP logging.

HomeEstate Realty — User Manual Section 4: User Roles

4. User Roles and Access Levels

Permissions and capabilities for each system user type

4.1 Administrator role_id = 1

The Administrator is the highest-level system user with full control over all platform operations.

  • View and manage all property listings across all statuses
  • Approve or reject agent-submitted property listings
  • Approve or reject registered agent accounts
  • Add new properties and agents directly to the system
  • Manage tour requests for admin-listed properties
  • Review, approve, or reject sale and rental verifications
  • Confirm or reject rental payment submissions
  • Process and record commission payments to agents
  • Access comprehensive reporting dashboard with export capabilities
  • Configure system settings (property types, amenities, specializations)
  • View and manage platform-wide notifications

4.2 Real Estate Agent role_id = 2

Agents are licensed real estate professionals who register through the system. Upon administrative approval, they gain full access to the agent portal.

  • Complete and maintain agent profile (license, specializations, bio, photo)
  • Create, edit, and manage property listings (subject to admin approval)
  • Upload, reorder, and delete property images and floor plans
  • Respond to tour requests (accept, reject, complete, cancel)
  • Submit sale and rental verifications for completed transactions
  • Record and submit monthly rental payments with supporting documents
  • Manage active leases (renew or terminate)
  • View earned commissions and download payment proof
  • Receive and manage agent-specific notifications

4.3 Public User / Client No Login Required

Public users are visitors who access the platform without authentication. No registration is required.

  • Browse the home page with featured properties and live statistics
  • Search and filter properties by city, type, price range, bedrooms, bathrooms, and more
  • View detailed property information with image galleries, floor plans, and amenities
  • Request property tours via the online scheduling form
  • Browse and view profiles of approved real estate agents
  • Like/unlike properties (tracked via browser session)
  • View the About page for company information

HomeEstate Realty — User Manual Section 5: Getting Started

5. Getting Started

Access, registration, login, two-factor authentication, and logout procedures

5.1 Accessing the System

To access HomeEstate Realty, open a supported web browser and navigate to the system URL:

http://localhost/capstoneSystem/user_pages/index.php

Public users are immediately presented with the home page without requiring login. Administrators and agents must navigate to the login page to authenticate:

http://localhost/capstoneSystem/login.php

5.2 Agent Registration

New real estate agents can create an account through the self-service registration process.

  1. Navigate to the login page and click "Register as an agent" below the login form.
  2. Complete all required fields: First Name, Last Name, Email, Phone Number (+639 format), Username, and Password (min 8 characters with letters and numbers).
  3. Click the "Register" button to submit the form.
  4. Upon success, you are redirected to the login page with the message: "Registration successful!"
  5. Log in and complete your agent profile. Your account must be approved by the administrator before full access is granted.

5.3 Logging In

Administrators and agents use the same login page to access the system.

  1. Navigate to the login page (login.php).
  2. Enter your Username and Password. Use the eye icon to toggle password visibility.
  3. Click the "Login" button. If credentials are valid, you are redirected to the 2FA page.
  4. If credentials are incorrect, an error message is displayed. For agents with incomplete profiles or pending approval, appropriate status messages are shown.

5.4 Two-Factor Authentication (2FA)

After successful credential verification, the system requires a one-time password (OTP) for enhanced security.

  1. The system automatically sends a 6-digit OTP code to your registered email address.
  2. Check your email inbox (and spam/junk folder) for the OTP from HomeEstate Realty.
  3. Enter the 6-digit code in the input field and click "Verify."
  4. Upon success: Admins are directed to the Admin Dashboard; Agents to the Agent Dashboard.
Important Notes

• The OTP code expires after 10 minutes. Click "Resend Code" to receive a new OTP.

• Maximum 3 attempts per OTP code. After exceeding attempts, request a new code.

• A countdown timer displays the remaining time before you may request a resend.

5.5 Logging Out

Click your profile name/avatar in the top-right corner, select "Logout" from the dropdown, and confirm the action in the modal dialog. The system also automatically logs out users after 30 minutes of inactivity.

6.3 Property Management

Access via sidebar "Properties". The page organizes listings into tabs: Pending Approved Rejected Pending Sold Sold Rented. Filters are available for property type, posted-by agent, and price range.

Approving a Property

  1. Navigate to the Pending Properties tab.
  2. Click "View Details" to review the property info, images, and amenities.
  3. Click the "Approve" button. The listing goes live and the agent is notified via email.

Rejecting a Property

  1. Click the "Reject" button on a pending property card.
  2. Enter a rejection reason in the modal dialog and click "Confirm Rejection."
  3. The property moves to the Rejected tab, and the agent is notified with the reason.

Adding a New Property (Admin Direct Listing)

Click "Add New Property" and complete the form: address fields (Street, City, Barangay, Province, ZIP — 4-digit PH format), property details (type, year built, Sq.Ft., lot size, beds, baths, parking), listing configuration (For Sale/For Rent, price, date), amenities checklist, property images (max 25 MB each — JPEG/PNG/GIF, drag-reorder), floor plans (by floor number), and description (max 2,000 characters). Admin-listed properties are visible immediately.

6.4 Agent Management

Access via sidebar "Agents". Tabs: Pending Approval Approved Needs Profile Rejected.

Approving an Agent

  1. Navigate to Pending Approval tab and click "View Profile" to review the agent's license, specializations, bio, experience, and photo.
  2. Click "Approve" to activate the agent's account. The agent receives an approval email and in-system notification.

Rejecting an Agent

  1. Click "Reject", enter a reason in the modal, and confirm.
  2. The agent is notified via email with the rejection reason.

Adding an Agent (Admin-Created)

Click "Add New Agent" and fill in: account info (name, email, phone, username, password), professional info (license number 5–30 chars, years of experience 0–99), at least one specialization, bio (max 2,000 chars), and profile picture (max 2 MB).

6.5 Tour Request Management

Access via sidebar "Tour Requests." This manages tours for admin-listed properties only. Tabs: All, Pending, Confirmed, Completed, Cancelled, Rejected, Expired. A property filter dropdown narrows results.

Accepting a Tour

  1. Navigate to Pending and click "Accept."
  2. The system checks for time conflicts (exact match + 30-minute travel buffer).
  3. If clear, the tour is confirmed and the visitor receives a confirmation email.

To complete a tour, click "Complete" on the Confirmed tab. To reject/cancel, enter a reason in the modal. Tours that pass their scheduled date automatically expire.

6.6 Sale Approvals

Access via sidebar "Sale Approvals." Tabs: All, Pending, Approved, Rejected. Each card shows the property, agent, sale price, buyer info, date, and attached documents.

Approving a Sale

  1. Review the sale verification details and supporting documents in the Pending tab.
  2. Click "Approve" and confirm in the modal.
  3. The system updates the verification to "Approved," marks the property as "Sold" (locked), creates a finalized sale record, logs a price history entry, and sends notifications to the agent and buyer.

To reject, enter a rejection reason — the property reverts to its prior status and the agent is notified.

⚡ Commission Note

Commission records are created as a separate step after sale approval — see Section 6.9 for commission processing.

6.7 Rental Approvals

Access via sidebar "Rental Approvals." Same tab layout as sales. Each card shows property, agent, monthly rent, security deposit, lease term, tenant details, and supporting documents.

Finalizing a Rental

  1. Review the rental verification in the Pending tab.
  2. Click "Approve" — in the modal, set the Commission Rate (%) per confirmed payment and optionally add admin notes.
  3. Click "Finalize Rental." The system creates an active lease with auto-calculated end date (start + term months), sends notifications to agent and tenant.

6.8 Rental Payment Management

Access via sidebar "Rental Payments." Section 1 shows active leases with KPIs (Active Leases, Pending Payments, Total Revenue). Section 2 shows payment submissions by status.

Confirming a Payment

  1. Review payment details, amount, period, and documents in the Pending tab.
  2. Click "Confirm." The payment is marked confirmed, and a commission record is auto-created if the lease has a commission rate > 0%. Commission = payment × (rate / 100).

6.9 Lease & Commission Management

Lease Management: From Rental Payments, click "View Payment History" on a lease card. The detailed page displays a hero section with property image, lease information (tenant, dates, term, commission rate, status), agent details, and a full payment history table with KPIs (confirmed/pending/rejected counts, total revenue, total commission).

Commission Processing (Sales)

Commission status lifecycle: Pending → Calculated → Processing → Paid (or Cancelled at any point).

  1. Navigate to the finalized sale record and click "Process Payment."
  2. Enter: Payment Method (Bank Transfer, GCash, Maya, Cash, Check, Other), Payment Reference, Notes, and upload Payment Proof (max 5 MB — JPG/PNG/WEBP/GIF/PDF).
  3. Click "Confirm Payment." The agent receives a notification with the reference and amount.

Rental commissions are auto-generated when rental payments are confirmed with a commission rate > 0%.

📊 Property Report

💰 Sales Report

Sale ID, Property, Final Price, Buyer, Agent, Date, Commission (amount/%), Status, Finalized By/Date. Totals row.

🏠 Rental Report

Rental ID, Property, Tenant, Monthly Rent, Lease Dates, Status, Agent, Payments Count, Revenue, Commission Rate/Total.

👥 Agent Performance

Agent Name, Specializations, Listings, Sales, Revenue, Commissions, Tours (Total/Completed), Status.

Tour Requests Report: Tour ID, Visitor, Property, Date/Time, Type, Status, Agent, Decision Reason. All reports support date range filters, status filters, text search, and pagination.

6.11 Notifications

Access via sidebar "Notifications." Filter by priority (Urgent/High/Normal/Low) and category tabs: All, Requests, Updates, Alerts, System. Each notification shows an icon, title, message, priority/category badges, timestamp, and read status. Actions: Mark as Read, Delete, Navigate to item, Mark All as Read, Delete All Read.

6.12 System Settings

Access via profile dropdown → "Settings." Three tabs for managing lookup values: Property Types (add/delete types like House, Condo), Amenities (add/delete like Swimming Pool, Garage), and Specializations (add/delete like Residential, Commercial). All operations use real-time AJAX.

6.13 Admin Profile

Access via profile dropdown → "Profile." View/update profile picture, license number, specialization, years of experience, and biography. Account info (name, email, username) is read-only.

7. Real Estate Agent Guide

Complete operations manual for licensed real estate agents

7.1 Agent Dashboard

After login and 2FA, agents are directed to the Agent Dashboard. It displays: a statistics grid (Total Listings, Tour Requests by status, Total Commission, Sales Completed), Upcoming Confirmed Tours (5), Pending Tour Requests, Recent Properties (4 cards), Top Performing Properties (3), and Quick Action buttons for uploading properties, managing tours, viewing commissions, and checking notifications.

7.2 Completing Your Agent Profile

After registration, agents must complete their profile before admin approval.

  1. Navigate to Profile via the navigation bar.
  2. Upload a Profile Picture (max 2 MB — JPEG/PNG/GIF/WEBP).
  3. Enter your License Number (5–30 characters, alphanumeric).
  4. Enter Years of Experience (0–99).
  5. Select at least one Specialization (Residential, Commercial, Rentals, etc.).
  6. Write a professional Biography (up to 2,000 characters).
  7. Click "Save Changes." The admin is automatically notified of your submission.

7.3 Property Management

Access via "Properties" in the navigation. Your portfolio is organized by status: Approved, Pending (grayed), Rejected (with reason shown), Pending Sold, Sold, and Rented.

Adding a New Property

  1. Click "Add New Property."
  2. Complete: Address (street, city, barangay, province, ZIP — 4-digit PH), property details (type, year built, beds/baths, sqft, parking), listing info (For Sale/Rent, price).
  3. For rentals: monthly rent, security deposit, lease term, furnishing, available-from date.
  4. Select amenities from the checklist. Upload images (max 25 MB each) and floor plans.
  5. Click "Submit Property." It enters Pending status awaiting admin approval.

Managing Images

In the property detail view: Upload via drag-and-drop, Reorder by dragging thumbnails (saved via AJAX), Delete by clicking the delete icon on hover. The first image by sort order is the primary listing photo.

Price History

The property detail view shows a Price History table with all price events (Listed, Price Change, Sold, Rented) including dates and values.

7.4 Tour Request Management

Access via "Tours" in navigation. Header shows counts by status. Filter by status tabs and property dropdown.

Accepting a Tour

  1. In the Pending tab, click "Accept."
  2. The system checks for time conflicts (same property exact time + 30-min buffer for different locations).

Click "Complete" after conducting the tour. Click "Reject" or "Cancel" with a reason — the visitor is notified. Unanswered tours past the scheduled date auto-expire.

7.5 Marking a Property as Sold

  1. Open the property detail view and click "Mark as Sold."
  2. Enter: Sale Price (> 0), Sale Date, Buyer Name, Buyer Email (optional), Additional Notes.
  3. Upload supporting documents (contracts, receipts — max 5 MB; PDF/DOCX/XLSX/JPG/PNG).
  4. Click "Submit Sale Verification." Property status changes to "Pending Sold."
  5. The admin reviews and approves/rejects. You receive a notification upon decision.

7.6 Marking a Property as Rented

  1. Open the property detail view and click "Mark as Rented."
  2. Enter: Monthly Rent, Security Deposit, Lease Start Date, Lease Term (months), Tenant Name/Email/Phone, Notes.
  3. Upload supporting lease documents (max 5 MB; PDF/DOCX/XLSX/JPG/PNG).
  4. Click "Submit Rental Verification." The admin is notified to review and finalize.

7.7 Rental Payment Recording

Access via "Rental Payments" in navigation. KPIs show pending/confirmed/rejected counts, total revenue, and commission earned.

  1. Click "Record Payment."
  2. Select the active lease, enter payment amount, date, period (start/end), and optionally upload a receipt (max 5 MB).
  3. Click "Submit Payment." It enters Pending status awaiting admin confirmation.
  4. Upon confirmation, a commission is auto-calculated if the lease has a commission rate > 0%.

7.8 Lease Management

Renew: Click "Renew Lease" — the end date extends by the original term. Status becomes "Renewed." Terminate: Click "Terminate Lease," enter a reason (required), and confirm. Property becomes available for new listings.

7.9 Commission Tracking

Access via "Commissions." Summary cards show Total Earnings, Paid, Pending, Calculated, Average Rate, and Cancelled. A monthly chart tracks earnings vs. payments. Tables list sales and rental commissions with status lifecycle: Pending → Calculated → Processing → Paid / Cancelled. When status is "Paid," download the payment proof file.

7.10 Notifications

The bell icon shows unread count. Full center: filter by All/Unread/Read and by category (Tours, Properties, Sales, Rentals). Types include: new tour, tour completed, property approved/rejected, sale/rental approved/rejected, commission paid, payment confirmed, and lease expiring/expired. Actions: Mark Read, Delete, View Details, Mark All Read.

HomeEstate Realty — User Manual Section 11: Best Practices

11. Best Practices & Usage Tips

Recommendations for efficient and secure system operation

11.1 For Administrators

  • Review pending items daily. Monitor the dashboard for pending approvals, agent registrations, and tour requests to keep the platform responsive.
  • Set commission rates carefully. The rate entered during rental finalization applies to all confirmed payments for the duration of the lease.
  • Provide clear rejection reasons. Reasons are sent to agents via email and help them understand corrections needed.
  • Monitor reports regularly. Use reports to track platform performance, agent productivity, and revenue. Export periodically.
  • Keep system settings current. Add new property types, amenities, and specializations as the market evolves.
  • Process commissions promptly. Upload payment proof when marking commissions as paid.

11.2 For Agents

  • Complete your profile immediately. A complete profile accelerates admin approval and builds client trust.
  • Upload high-quality images. Use well-lit, high-resolution photos. Set the most appealing image as the first in sort order.
  • Write detailed descriptions. Include neighborhood info, nearby amenities, transport access, and unique features.
  • Respond to tours within 24 hours. Pending tours that pass their date auto-expire, potentially losing client interest.
  • Submit complete documentation. Include all supporting documents with sale and rental verifications to speed admin review.
  • Record rental payments regularly. Submit monthly records promptly with receipts for accurate commission tracking.
  • Monitor notifications actively. The bell icon badge updates automatically — do not miss important approvals or tour requests.

11.3 For Public Users

  • Use filters strategically. Combine city, type, price range, and bed/bath filters to find the most relevant listings.
  • Provide accurate contact info in tour requests. Confirmations are sent to the email you provide.
  • Schedule tours in advance. Select future dates to give agents time to respond.
  • Browse agent profiles. Check specializations and experience before requesting tours.
  • Like properties for later. Likes help you track interests (note: stored in browser — clearing data resets them).

11.4 Security Recommendations

🔒 Security Best Practices

Strong passwords — Use at least 8 characters with letters and numbers. Avoid personal information.

Protect your OTP — Never share 2FA codes. HomeEstate staff will never ask for your OTP.

Log out after sessions — Especially on shared/public computers, always use the Logout function.

Verify email links — Ensure notification emails originate from the official system before clicking.

Keep contact info current — Update your email and phone to receive security codes and notifications.
